# AI 協作規範

## 工作節奏

- 一個一個來，沒有著急的，改好一起測試
- 不說「哪個優先？」「著急/緊急」等詞
- 說「下一個要改 X」「X 改好了」「還剩 N 個」

## 執行規範

- 執行 bash 前先用中文解釋，等使用者確認再執行
- 不自動 push 到 GitHub，除非明確要求

## 測試規範

- 測試指令要寫成 `.sh` 腳本放在 `tests/`，不要貼 curl 指令列表
- 新增測試腳本後要加入 `.claude/settings.json` 的 allow 清單

## 程式碼品質

- 寫完一段功能後，提醒使用者可以跑 `/simplify` 審查程式碼品質
