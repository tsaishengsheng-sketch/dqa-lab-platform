# AI 協作規範

## 工作節奏

- 先整理清單，再做「一次批次更新」，避免零碎改動反覆打斷
- 回報格式固定：`已完成` / `進行中` / `剩餘項目`
- Demo 前以阻塞問題優先（例如登入、核心流程、PDF）

## 執行規範

- 低風險命令可直接執行並回報結果
- 高風險或破壞性命令（刪檔、重設、對外部署）先確認
- 不自動 push 到 GitHub，除非明確要求

## 測試規範

- 測試指令要寫成 `.sh` 腳本放在 `tests/`，不要貼 curl 指令列表
- 新增測試腳本後要加入 `.claude/settings.json` 的 allow 清單

## 程式碼品質

- 優先「最小必要修改」，避免一次引入過多新結構
- 改動前先說明範圍，改動後附最短驗證結果
